## Executive Summary

SubTrack 是一款 iOS 原生订阅管理应用，帮助用户掌控不断膨胀的数字订阅支出。

核心主张：**"知道自己在花什么钱，不再为不用的服务付账。"**

SubTrack 不只是一个记账工具。两个核心差异化功能——**AI 智能录入**和**社区评分**——将产品从"个人工具"升级为"订阅决策平台"。社区评分是 SubTrack 的主要商业化路径，也是建立长期数据壁垒的核心资产。

## 一、市场背景

### 1.1 订阅经济的结构性问题

- 全球数字订阅市场规模 2025 年超过 6500 亿美元，年增长率约 18%
- 西方市场调研：普通用户平均订阅 8-12 个服务，但能准确说出月均支出的人不足 20%
- 中国市场：流媒体 + 效率工具 + 云存储的三层叠加订阅正在快速普及
- "订阅疲劳"成为新词：用户厌倦了续费提醒被忽视、不知不觉被扣款

### 1.2 现有竞品的局限

| 竞品 | 优势 | 核心局限 |
|:-----|:-----|:---------|
| iOS 设置 > 订阅 | 零门槛，官方数据 | 只显示 Apple 生态内订阅，不含非 IAP 服务 |
| Subtrack (国外同名) | 简洁 UI | 无 AI 功能，无社区维度，无中文支持 |
| Bobby | 手动录入流畅 | 无分析能力，无社区，无 AI |
| 支付宝账单 / 微信账单 | 数据完整 | 不提供订阅维度的聚合分析，不可管理 |

### 1.3 SubTrack 的差异化定位

```
                    AI 能力
                      ▲
                      │
         SubTrack  ●  │
                      │
                      │
    Bobby ●           │
                      │
    ──────────────────┼──────────────────→ 社区数据
                      │           （评分/评价/决策参考）
    iOS订阅 ●          │
```

SubTrack 是唯一同时具备 **AI 录入能力** 和 **社区决策维度** 的订阅管理工具。

## 二、目标用户

### Persona A · 小李 — 订阅焦虑型（核心用户，占比估计 55%）

- **年龄**：24-28 岁，互联网/科技行业从业者
- **订阅量**：8-15 个，每月支出约 ¥200-500
- **核心痛点**：订阅了很多，不清楚在用哪些，偶尔发现为已弃用的服务白白续费
- **关键动机**：省钱 + 掌控感
- **使用路径**：注册 → 快速录入全部订阅 → 依赖提醒功能 → 偶发查看支出报告

### Persona B · 王姐 — 精打细算型（深度用户，占比估计 30%）

- **年龄**：29-38 岁，自由职业者 / 创业者
- **订阅量**：5-10 个，但注重 ROI
- **核心痛点**：知道有支出但不知道哪笔"值"，想要依据做取舍
- **关键动机**：优化支出结构，不浪费
- **使用路径**：注册 → 认真查看 Insights → 关注 AI 建议 → 参考社区评分做决策

### Persona C · 张哥 — 效率优先型（获客型用户，占比估计 15%）

- **年龄**：26-35 岁，产品经理 / 工程师
- **订阅量**：10+ 个，录入意愿强但嫌麻烦
- **核心痛点**：不想手动逐个输入
- **关键动机**：AI 扫描功能是他下载 App 的唯一理由
- **使用路径**：下载 → 用 AI 扫描录入 → 留存取决于 AI 准确率

### 4.1 AI Smart Scanner（AI 智能录入）

**功能定位**

> 将"手动填写 8 个字段"简化为"拍一张照片，确认一下"。

录入订阅是产品的第一步，也是流失风险最高的环节。AI Scanner 的目标是将录入时间从约 60 秒/个降低到约 10 秒/个，将张哥这类用户的转化率提升至少 40%。

**用户流程**

```
┌──────────┐     ┌──────────┐     ┌──────────┐
│  SELECT  │ ──→ │ AI SCAN  │ ──→ │ CONFIRM  │
│ 选择图片  │     │ AI 识别中 │     │ 确认/修改 │
└──────────┘     └──────────┘     └──────────┘
```

- **Step 1**：选择输入方式（拍照 / 相册），选择来源类型（App Store 订单 / 邮件收据 / 银行账单）
- **Step 2**：扫描动画 2-5 秒，AI 执行 OCR + 结构化提取
- **Step 3**：展示提取结果，高置信度字段正常显示，低置信度字段橙色高亮提示用户核实

**置信度策略**

| 置信度 | UI 表现 | 用户操作 |
|:-------|:--------|:---------|
| ≥ 90% | 白色正常显示 | 可选修改 |
| 70–89% | 橙色高亮 ⚠️ | 建议核实 |
| < 70% | 留空，显示"请手动输入" | 必须填写 |

**技术选型**
- OCR：Apple Vision Framework（端侧优先，降低延迟和隐私风险）
- 结构化提取：Claude API（prompt 包含来源类型和服务数据库）
- 服务数据库：约 500 个常见订阅服务（名称、别名、图标、类别、价格区间）

**当前 App 内展示策略**

添加订阅流程保留完整的三步框架。Step 1（选择方式）和 Step 2（AI 扫描）以精心设计的**功能预告界面**展示：
- 清晰描述 AI 扫描的工作方式和三种识别来源
- 展示置信度评分系统的 UI 示意
- 底部提供"已了解，手动填写详情 →"按钮，流畅衔接现有功能
- **不是死路，而是一个可操作的预告**

目标：让用户了解功能、建立期待，同时不阻塞核心使用流程。

**预期上线**：V1.0（参见路线图第六节）

### 4.2 AI Suggestions（AI 省钱建议）

**功能定位**

> 不只帮用户"看"订阅，还帮用户"省"钱。

被动的订阅列表只解决了"我有什么"，AI Suggestions 解决的是"我应该怎么做"。这是 SubTrack 从工具升级为顾问的关键功能，也是付费转化（SubTrack Pro）的核心价值驱动。

**建议类型一：重复订阅检测**

检测功能存在重叠的订阅组合：
- **包含关系**（如 Apple Music ⊂ Apple One）：明确建议取消前者，显示可节省金额
- **同类竞品**（如 Netflix ≈ Disney+）：温和提问语气，不做强制建议

**建议类型二：年付切换建议**

触发条件：月付订阅 + 服务有年付选项 + 用户已连续订阅 ≥ 3 个月
计算：节省金额 = (月付 × 12) - 年付价格，仅当节省 > 月费 5% 时生成建议

**未来可扩展的建议类型**
- 价格上涨预警（V2.0）
- 长期未使用提醒，关联 Screen Time API（V2.0）
- 更优套餐推荐（V2.0）
- 家庭方案合并建议（V2.5）

**当前 App 内展示策略**

Insights 页面 AI Suggestions 区块展示精心设计的预告模块，包含：
- 两类建议的真实 UI 示意（重复检测卡片 + 年付建议卡片的视觉原型）
- 功能说明文案
- "上线后为你自动分析"的注册期待

**预期上线**：V1.0

### 4.3 Community Ratings & Reviews（社区评分）

**战略定位**

这是 SubTrack **最重要的长期资产和主要商业化路径**，不是附加功能。

一个普通的订阅管理 App 帮用户"记录"，Community Ratings 让 SubTrack 帮用户"决策"——而且这个决策能力随着社区规模增长而不断增强，形成竞争壁垒。

**功能定义**

每个订阅服务有一个社区评分页面，包含：

| 数据维度 | 内容 |
|:---------|:-----|
| **综合评分** | 1-5 星，展示评分分布直方图 |
| **评分人数** | SubTrack 用户中订阅该服务的人数 |
| **标签云** | 高频出现的用户标签（如"物有所值""内容丰富""性价比差""经常涨价"） |
| **取消率** | 过去 12 个月内取消该服务的用户比例（匿名聚合） |
| **留存曲线** | 首次订阅后 3/6/12 个月的留存率（匿名聚合） |
| **用户评价** | 文字评价（Pro 功能，见商业模式） |
| **价格历史** | 该服务在 SubTrack 用户中记录到的历史价格变化 |

**评分提交流程**

- 用户订阅某服务 ≥ 1 个月后，可提交评分
- 取消订阅时，系统询问"能说说取消的原因吗？"（可选，1 题单选 + 可选文字）
- 每用户每服务只能评一次，可修改
- 评分权重：取消时填写的原因权重 ×1.5（信号更强）

**内容质量控制**

| 机制 | 说明 |
|:-----|:-----|
| 门槛验证 | 必须在 SubTrack 内有该订阅记录才能评分 |
| 异常检测 | 单日大量相似评分触发人工审核 |
| 举报机制 | 用户可举报不当评价 |
| 最小样本 | 评分 < 5 人时只显示"评价人数不足"，不显示分数 |

**商业模式（见第五节详述）**

- 免费用户：看综合评分 + 前 3 个标签
- Pro 用户：看完整评价文字 + 取消率 + 留存曲线 + 价格历史

**当前 App 内展示策略**

订阅详情页社区评分区块升级为有存在感的**预告模块**：
- 展示模糊化处理的示例评分界面（评星 + 标签 + 人数）
- 说明文案：展示 SubTrack 社区将为用户提供什么价值
- "上线后通知我"按钮（本地存储兴趣记录，量化需求热度）
- 视觉权重与其他功能区块对等，体现战略地位

## 五、商业模式

### 5.1 Freemium 层级结构

| 功能 | 免费版 | SubTrack Pro |
|:-----|:------:|:------------:|
| 订阅管理（增删改查） | ✅ | ✅ |
| 仪表盘 & Insights | ✅ | ✅ |
| 推送提醒 | ✅ | ✅ |
| 数据导出（CSV/PDF） | ✅ | ✅ |
| AI 录入（基础扫描） | ✅ | ✅ |
| 社区评分（综合分 + 前 3 标签） | ✅ | ✅ |
| AI 建议（基础检测） | ✅ | ✅ |
| **社区完整评价文字** | ❌ | ✅ |
| **取消率 & 留存曲线** | ❌ | ✅ |
| **价格历史图表** | ❌ | ✅ |
| **AI 建议优先推送** | ❌ | ✅ |
| **无广告** | ❌ | ✅ |

### 5.2 定价策略

- **月付**：¥18 / 月（对标 Spotify 学生版价位，可接受）
- **年付**：¥168 / 年（= ¥14/月，省 22%；策略上引导年付）
- **促销策略**：订阅满 5 个服务时，App 内推送"你的订阅已值得 Pro 分析，首月 ¥9"

### 5.3 收入来源

**主营收入：SubTrack Pro 订阅**
- 目标：DAU 的 5% 付费转化率
- 核心转化场景：用户查看某服务社区评分时，看到"完整评价需要 Pro"

**次要收入：联盟推广（Affiliate）**
- 场景：社区评分高的服务 + 用户当前未订阅 → 展示"试用优惠"入口
- 原则：只推荐社区评分 ≥ 4.0 的服务，维护用户信任
- 预估：单次转化收入约 ¥15-50（订阅服务推广佣金）

**长期收入：B2B 匿名数据（V2.5 探索）**
- 价值：订阅服务商愿意付费了解"为什么用户取消订阅"
- 原则：严格匿名聚合，绝不出售个人数据；隐私政策明确告知

### 5.4 增长飞轮

```
更多用户
   ↓
更多评分数据
   ↓
更准确的社区评分
   ↓
更好的 AI 建议（基于真实取消率）
   ↓
更强的用户决策价值
   ↓
更多用户（口碑 + App Store 评分）
```

## 八、成功指标

### 北极星指标

**"每月至少打开 App 一次、且有 ≥3 个活跃订阅的用户比例"**

> 选择依据：这个指标同时验证了"用户觉得 App 有价值（留存）"和"用户真的在用它管理订阅（核心行为）"。两者缺一不可的产品才是真正有价值的产品。

### 各版本关键指标

| 版本 | 指标 | 目标 | 测量方式 |
|:-----|:-----|:----:|:---------|
| MVP (当前) | 首次打开后完成 ≥1 个订阅录入的用户比例 | > 60% | 本地埋点 |
| MVP (当前) | 7 日留存率 | > 40% | 本地埋点 |
| V1.0 | AI Scanner 识别准确率（必填字段全部正确） | > 80% | 后台日志 |
